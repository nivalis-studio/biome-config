# Biome Insights (artifact-first v1)

This is a deterministic first version of cross-repo Biome analytics.

## Goals

- run Biome with JSON reporter in each repo CI
- normalize diagnostics into a stable schema
- upload artifacts from each run
- aggregate artifacts later into markdown reports

This phase intentionally avoids AI-generated recommendations.

## Reusable workflow

Use the reusable workflow from this repository in each project:

```yaml
name: Biome Insights

on:
  pull_request:
  push:
    branches: [main]

jobs:
  biome-insights:
    uses: nivalis-studio/biome-config/.github/workflows/biome-insights-artifact.yaml@main
    with:
      shared-config-version: '@nivalis/biome-config@2.4.9'
```

Optional:

- `local-config-hash`: set this from your repo if you need drift tracking.

The reusable workflow uses `biome ci --reporter=json` (not `biome check`) so CI behavior matches Biome's recommended command semantics while still producing machine-readable output.

## Artifact schema

Artifacts are generated by:

- `scripts/biome-insights/normalize.mjs`

Output file: `.biome-insights/biome-normalized.json`

Top-level shape:

- `schemaVersion`
- `generatedAt`
- `run` (repo metadata and config metadata)
- `summary` (errors, warnings, infos, fixable)
- `diagnostics` (normalized findings)

Diagnostic fields:

- `category`, `domain`, `rule`, `severity`
- `filePath`, `pathFamily`, `line`, `column`
- `message`, `fixable`, `fingerprint`

`fingerprint` is deterministic from category, path, location, and normalized message.

## Local usage

Normalize one run:

```sh
pnpm insights:normalize --input ./biome-raw.json --output ./biome-normalized.json --repo nivalis-studio/example
```

Generate aggregate markdown from a directory of normalized JSON artifacts:

```sh
pnpm insights:summary --input-dir ./artifacts --output ./artifacts/summary.md
```

The summary includes:

- top noisy rules by repo
- autofix rate by rule
- config drift matrix (when hashes are provided)
- weekly trend
